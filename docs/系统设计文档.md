# LoyaltyLink 企业联盟通用积分系统 - 系统设计文档

## 一、引言

### 1.1 文档目的
本文档详细描述 LoyaltyLink 企业联盟通用积分系统的技术架构设计，包括系统架构、模块划分、数据结构、接口设计、安全性考虑及部署方案，为开发团队提供技术实现指导。

### 1.2 项目背景
LoyaltyLink 是基于以太坊 ERC-20 标准构建的企业联盟积分系统，旨在解决传统积分体系的三大痛点：
- 积分价值低、兑换门槛高、容易过期
- 商家自建积分系统成本高、用户粘性低
- 缺乏跨商家流通能力，形成数据孤岛

- 《LoyaltyLink —— 基于ERC-20的企业联盟通用积分系统.md》（需求分析报告）
- 《环境配置-已验证.md》（环境搭建指南）


### 2.2 架构特点

| 特性 | 描述 | 实现方式 |
| :--- | :--- | :--- |
| **去中心化** | 积分作为 ERC-20 代币存储在链上，用户完全掌控资产 | 基于以太坊私有链（Ganache） |
| **透明性** | 所有交易记录公开可查，无法篡改 | 区块链账本特性 |
| **权限控制** | 商家和管理员角色分离，操作权限严格控制 | OpenZeppelin Ownable + 自定义 modifier |
| **安全暂停** | 管理员可在紧急情况下暂停所有积分操作 | OpenZeppelin Pausable |
| **可扩展性** | 支持动态添加/移除商家，无需重新部署合约 | `addMerchant`/`removeMerchant` 函数 |
| **积分过期** | 可设置积分有效期，超期未活动积分自动销毁 | `expiryTime` + `checkAndBurnExpired()` |
| **联盟结算** | 商家间可直接转账积分用于联盟内结算 | `settle()` 函数 |
| **灵活汇率** | 各商家可自定义积分兑换比例 | `exchangeRates` + `setExchangeRate()` |

## 三、模块设计

### 3.1 智能合约模块

#### 3.1.1 核心合约结构

**合约名称**: `LoyaltyToken`

**继承关系**:
- `ERC20` - 标准代币功能
- `Ownable` - 所有权管理
- `Pausable` - 紧急暂停机制

**核心状态变量**:

| 变量名 | 类型 | 可见性 | 描述 |
| :--- | :--- | :--- | :--- |
| `merchants` | `mapping(address => bool)` | `public` | 商家白名单映射 |
| `expiryTime` | `uint256` | `public` | 积分有效期（秒），默认 365 天，0=永不过期 |
| `lastActivity` | `mapping(address => uint256)` | `public` | 用户最后活动时间戳 |
| `lastFaucetClaim` | `mapping(address => uint256)` | `public` | 水龙头上次领取时间 |
| `exchangeRates` | `mapping(address => uint256)` | `public` | 商家积分汇率映射 |

**核心事件**:

| 事件名 | 参数 | 触发时机 |
| :--- | :--- | :--- |
| `MerchantAdded` | `address indexed merchant` | 商家添加成功 |
| `MerchantRemoved` | `address indexed merchant` | 商家移除成功 |
| `Reward` | `address indexed fromMerchant, address indexed toUser, uint256 amount` | 积分发放成功 |
| `Spend` | `address indexed fromMerchant, address indexed fromUser, uint256 amount` | 积分消费成功 |
| `TokensRedeemed` | `address indexed user, uint256 amount` | 用户自主销毁积分 |
| `ContractPaused` | `address indexed by` | 合约暂停 |
| `ContractUnpaused` | `address indexed by` | 合约恢复 |
| `PointsExpired` | `address indexed user, uint256 amount` | 过期积分被销毁 |
| `Settlement` | `address indexed fromMerchant, address indexed toMerchant, uint256 amount` | 商家间结算 |
| `FaucetClaim` | `address indexed user, uint256 amount` | 水龙头领取 |
| `ExchangeRateSet` | `address indexed merchant, uint256 rate` | 商家设置汇率 |

**核心函数**:

| 函数名 | 权限 | 功能描述 |
| :--- | :--- | :--- |
| `addMerchant(address merchant)` | `onlyOwner` | 将地址添加为商家 |
| `removeMerchant(address merchant)` | `onlyOwner` | 移除商家权限 |
| `pause()` | `onlyOwner` | 暂停所有积分操作 |
| `unpause()` | `onlyOwner` | 恢复合约运行 |
| `setExpiryTime(uint256)` | `onlyOwner` | 设置积分有效期 |
| `reward(address user, uint256 amount)` | `onlyMerchant` | 商家向用户发放积分（mint） |
| `spend(address user, uint256 amount)` | `onlyMerchant` | 商家从用户账户扣除积分（burn） |
| `settle(address toMerchant, uint256 amount)` | `onlyMerchant` | 商家间结算转账 |
| `setExchangeRate(uint256 rate)` | `onlyMerchant` | 商家设置积分汇率 |
| `redeemTokens(uint256 amount)` | 任何人 | 用户自主销毁积分兑换奖品 |
| `checkAndBurnExpired(address user)` | 任何人 | 检查并销毁过期积分 |
| `faucet()` | 任何人 | 领取 5 LYL 测试积分（冷却 1 天） |
| `transfer(address, uint256)` | 任何人 | ERC-20 转账（暂停保护） |
| `transferFrom(address, address, uint256)` | 授权用户 | ERC-20 代理转账（暂停保护） |

#### 3.1.2 合约交互流程图

**积分发放流程（Reward）**:
```
用户消费 → 商家确认金额 → 商家调用 reward(user, amount) → MetaMask签名 → 交易上链 → 用户余额增加
```

**积分消费流程（Spend）**:
```
用户兑换商品 → 商家确认扣减数量 → 商家调用 spend(user, amount) → 合约校验余额 → 扣减积分 → 交易完成
```

**用户转账流程（Transfer）**:
```
用户A发起转账 → 调用 transfer(to, amount) → MetaMask签名 → 交易上链 → 双方余额更新
```

## 六、安全性设计

### 6.1 合约安全

#### 6.1.1 权限控制

| 安全措施 | 实现方式 | 保护目标 |
| :--- | :--- | :--- |
| 管理员权限 | `onlyOwner` modifier | 防止非管理员添加/移除商家、暂停合约、修改有效期 |
| 商家权限 | `onlyMerchant` modifier | 防止普通用户发放/扣减积分、结算、设置汇率 |
| 余额校验 | `require(balanceOf(user) >= amount)` | 防止扣减/结算超过账户余额 |
| 暂停保护 | `whenNotPaused` modifier | 紧急情况下冻结所有 on-chain 操作 |
| 冷却限制 | `FAUCET_COOLDOWN` 时间间隔检查 | 防止水龙头被滥用 |
| 过期机制 | `expiryTime` + `lastActivity` 双重校验 | 自动清理长期不活动的积分 |

#### 6.1.2 重入攻击防护
- 使用 OpenZeppelin 标准库，内部函数 `_mint`/`_burn` 已受保护
- 所有外部调用在状态更新之后执行

#### 6.1.3 整数溢出防护
- Solidity 0.8.x 版本内置溢出检查，无需额外处理

### 6.2 前端安全

#### 6.2.1 钱包连接验证
```javascript

```

```
   - 端口: 7545
   - 链 ID: 1337
   - 助记词: maid notable twist mutual dune speed come dolphin wet gaze scout sort
```
