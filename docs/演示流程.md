# LoyaltyLink 项目演示流程

## 角色与账户规划

| 角色 | Ganache 账户 | 用途 |
|:---|:---|:---|
| 管理员 (Owner) | Account 0 | 部署合约、添加/移除商家 |
| 商家 (Merchant) | Account 1 | 发放积分 (reward)、扣除积分 (spend) |
| 用户 A | Account 2 | 持有积分、转账 |
| 用户 B | Account 3 | 接收转账 |

---

## 环境准备

确保以下服务已启动：

| 服务 | 说明 |
|:---|:---|
| Ganache | 本地以太坊测试网，端口 `7545`，Chain ID `1337` |
| 前端服务器 | `http-server` 或 `serve`，端口 `3000` |
| MetaMask | 浏览器插件，连接 Ganache 网络 |

**助记词**：`maid notable twist mutual dune speed come dolphin wet gaze scout sort`

> 所有成员使用相同助记词启动 Ganache，可获得相同的 10 个账户（每个 100 ETH）。

---

## 步骤一：启动服务

### 1.1 启动 Ganache

```bash
# 方式一：Ganache GUI
# 打开 Ganache，确保端口为 7545，使用上述助记词

# 方式二：Ganache CLI
npx ganache --port 7545 --chain.chainId 1337 \
  --wallet.mnemonic "maid notable twist mutual dune speed come dolphin wet gaze scout sort"
```

### 1.2 部署合约

```bash
# 编译 + 部署（自动更新前端合约地址）
npm run deploy:full
```

记录输出的合约地址，确认 `frontend/app.js`、`frontend/merchant.js`、`frontend/admin.js` 中的 `contractAddress` 已自动更新。

### 1.3 启动前端

```bash
npx http-server frontend/ -p 3000 -c-1
```

> `-c-1` 禁用缓存，确保每次加载最新代码。

### 1.4 配置 MetaMask

1. 打开浏览器 MetaMask 插件
2. **添加自定义网络**：
   - 网络名称：`Ganache Local`
   - RPC URL：`http://127.0.0.1:7545`
   - Chain ID：`1337`
   - 货币符号：`ETH`
3. **导入至少 3 个账户**（使用 Ganache 提供的私钥）：
   - Account 0 → 管理员
   - Account 1 → 商家
   - Account 2 → 用户 A

---

## 步骤二：演示流程

### 2.1 管理端 — 添加商家

1. 打开 `http://localhost:3000/admin.html`
2. MetaMask 切换到 **Account 0（管理员）**
3. 点击 **"连接 MetaMask"**
4. 确认页面显示：
   - Owner 地址 = Account 0 地址
   - "管理员状态: 是管理员"
5. 在 **"添加商家"** 区域：
   - 输入 Account 1（商家）的地址
   - 点击 **"添加商家"**
6. MetaMask 弹出确认 → 确认交易
7. 页面显示 **"商家添加成功"**，白名单中出现 Account 1

**预期结果：** Account 1 被添加为商家，白名单列表更新。

---

### 2.2 商户端 — 发放 LYL 积分

1. 打开 `http://localhost:3000/merchant.html`
2. MetaMask 切换到 **Account 1（商家）**
3. 点击 **"连接 MetaMask"**
4. 确认页面显示 **"商家状态: 是商家"**
5. 在 **"发放 LYL 积分"** 区域：
   - 输入 Account 2（用户 A）的地址
   - 输入数量：`100`
   - 点击 **"发放 LYL"**
6. MetaMask 弹出确认 → 确认交易
7. 页面显示 **"LYL 积分发放成功！"**

**预期结果：** 用户 A 收到 100 LYL，商户 LYL 余额不变（reward 是 mint，不消耗商户余额）。

---

### 2.3 用户端 — 查看余额和转账

1. 打开 `http://localhost:3000/index.html`
2. MetaMask 切换到 **Account 2（用户 A）**
3. 点击 **"连接 MetaMask"**
4. 确认页面显示 **"LYL 余额: 100 LYL"**
5. 在 **"转账 LYL"** 区域：
   - 输入 Account 3（用户 B）的地址
   - 输入数量：`30`
   - 点击 **"转账 LYL"**
6. MetaMask 弹出确认 → 确认交易
7. 页面显示 **"LYL 转账成功！"**，用户 A 余额变为 70 LYL

**预期结果：** 用户 A 余额 70 LYL，用户 B 余额 30 LYL。

---

### 2.4 商户端 — 扣除 LYL 积分（模拟积分消费）

1. 回到商户页面 `http://localhost:3000/merchant.html`
2. 确保 MetaMask 仍为 **Account 1（商家）**
3. 在 **"扣除 LYL 积分"** 区域：
   - 输入 Account 2（用户 A）的地址
   - 输入数量：`20`
   - 点击 **"扣除 LYL"**
4. MetaMask 弹出确认 → 确认交易
5. 页面显示 **"LYL 积分扣除成功！"**
6. 在 **"查询任意地址积分余额"** 区域：
   - 输入 Account 2 的地址
   - 点击 **"查询余额"**
   - 确认显示 50 LYL（100 - 30 转账 - 20 扣除）

**预期结果：** 用户 A 余额从 70 减为 50 LYL，扣除的 20 LYL 被销毁（burn）。

---

### 2.5 管理端 — 验证商家管理

1. 回到管理页面 `http://localhost:3000/admin.html`
2. 在 **"查询指定地址是否为商家"** 区域：
   - 输入 Account 1 地址 → 点击查询 → 显示 "是商家"
   - 输入 Account 2 地址 → 点击查询 → 显示 "不是商家"
3. 在 **"移除商家"** 区域：
   - 输入 Account 1 地址
   - 点击 **"移除商家"**
4. MetaMask 确认后，白名单清空
5. 切换到商户页面验证：Account 1 不再显示为商家

**预期结果：** 商家权限可动态管理，非管理员无法操作。

---

## 演示要点总结

| 步骤 | 页面 | 操作 | 展示的核心能力 |
|:---|:---|:---|:---|
| 2.1 | 管理端 | 添加商家 | 管理员权限控制、商家白名单 |
| 2.2 | 商户端 | 发放积分 | 商家 mint 积分、ERC-20 增发 |
| 2.3 | 用户端 | 查看余额 + 转账 | 用户自主管理积分、P2P 转账 |
| 2.4 | 商户端 | 扣除积分 + 查询余额 | 商家 burn 积分（消费场景）、跨账户查询 |
| 2.5 | 管理端 | 查询/移除商家 | 权限验证、动态白名单管理 |

**关键机制：**
- **Reward (mint)**：商家发放积分 = 铸币，总供应量增加
- **Spend (burn)**：商家扣除积分 = 销毁，总供应量减少（模拟用户用积分兑换商品）
- **Transfer**：用户间转账 = ERC-20 标准转账，总供应量不变
- 所有交易可在 MetaMask 历史记录查看，区块数据可在 Ganache 中查验
