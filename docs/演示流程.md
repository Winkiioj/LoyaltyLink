## 步骤二：演示流程

### 2.1 管理端 — 验证商家白名单

> 💡 一键启动已自动注册 Account 1 为商家。此步骤改为验证 + 演示添加/移除。

1. 打开 `http://localhost:3000/admin.html`
2. MetaMask 切换到 **Account 0（管理员）**
3. 点击 **"连接 MetaMask"**
4. 确认页面显示：
   - Owner 地址 = Account 0 地址
   - "管理员状态: 是管理员"
   - 白名单中已有 Account 1（商家名称如已设置会显示）
5. 在 **"查询指定地址是否为商家"** 区域：
   - 输入 Account 1 地址 → 显示 "是商家"
   - 输入 Account 2 地址 → 显示 "不是商家"
6. （可选）演示添加额外商家：输入 Account 4 地址 → 点击 **"添加商家"**
7. MetaMask 弹出确认 → 确认交易，白名单更新

**预期结果：** Account 1 已预注册为商家，白名单查询正常。演示后可移除 Account 4 恢复初始状态。

---

### 2.1 商户端 — 发放 LYL 积分

1. 打开 `http://localhost:3000/merchant.html`
2. MetaMask 切换到 **Account 1（商家）**
3. 点击 **"连接 MetaMask"**
4. 确认页面显示 **"商家状态: 是商家"**
5. 在 **"手动操作"** → **"发放 LYL 积分"** 区域：
   - 输入 Account 2（用户 A）的地址
   - 输入数量：`1000`
   - 点击 **"发放 LYL"**
6. MetaMask 弹出确认 → 确认交易
7. 页面显示 **"LYL 积分发放成功！"**

**预期结果：** 用户 A 累计 6000 LYL（初始 5000 + 本次 1000），商户 LYL 余额不变（reward 是 mint，不消耗商户余额）。

---

### 2.2 用户端 — 查看余额和转账

1. 打开 `http://localhost:3000/index.html`
2. MetaMask 切换到 **Account 2（用户 A）**
3. 点击 **"连接 MetaMask"**
4. 确认页面显示 **"LYL 余额: 6000 LYL"**（初始 5000 + 步骤 2.1 发放 1000）
5. 在 **"转账 LYL"** 区域：
   - 输入 Account 4（用户 B）的地址
   - 输入数量：`300`
   - 点击 **"转账 LYL"**
6. MetaMask 弹出确认 → 确认交易
7. 页面显示 **"LYL 转账成功！"**，用户 A 余额变为 5700 LYL

**预期结果：** 用户 A 余额 5700 LYL，用户 B 余额 2300 LYL（初始 2000 + 收到 300）。

---

### 2.3 积分商城 — 浏览商品并下单

1. 打开 `http://localhost:3000/shop.html`
2. MetaMask 使用 **Account 2（用户 A）**
3. 点击 **"连接 MetaMask"**
4. 浏览商品列表（珍珠奶茶、咖啡、打印、洗衣、水果茶、装订）
5. 点击奶茶的 **"购买"** → 页面显示"订单已创建"
6. 向下滚动，查看 **"我的订单"** 列表：
   - 确认出现刚下的订单，状态为"待确认"
   - 显示商品名、商家、价格、时间

**预期结果：** 订单记录保存到 localStorage，状态 pending，等待商家确认。此步骤不触发链上交易。

---

### 2.4 积分兑换 — 使用 LYL 兑换奖品

1. 打开 `http://localhost:3000/redeem.html`
2. MetaMask 使用 **Account 2（用户 A）**
3. 点击 **"连接 MetaMask"**，确认余额 ~5700 LYL
4. 浏览可兑换奖品（优惠券、奶茶券、笔记本等）
5. 点击 **"¥5 优惠券"**（500 LYL）的兑换按钮
6. 弹出确认对话框 → 点击 **"确认兑换"**
7. MetaMask 弹出 spend() 交易确认 → 确认交易
8. 页面显示"兑换成功"，余额减少 500 LYL（约 5200 LYL 剩余）
9. 查看 **"我的兑换记录"**，确认出现新记录

**预期结果：** 用户 A 余额从 ~5700 减为 ~5200 LYL。兑换记录保存到 localStorage，包含交易哈希。

### 2.5 商户端 — 处理商城订单

1. 打开 `http://localhost:3000/merchant.html`
2. MetaMask 切换到 **Account 1（商家）**
3. 点击 **"连接 MetaMask"**
4. 在 **"待处理订单（商城）"** 区域：

5. MetaMask 弹出 reward() 交易确认 → 确认交易
6. 页面显示"积分发放成功"，订单状态变为"已确认"
7. 在 **"兑换请求"** 区域：
   - 确认显示用户 A 的 ¥5 优惠券兑换记录（500 LYL，已兑换）

**预期结果：** 用户 A 收到 150 LYL（余额 ~5350 LYL），订单状态更新。兑换记录可见。

## 步骤三（可选）：紧急暂停与恢复

演示 LoyaltyToken 合约的 Pausable 安全机制——管理员可在紧急情况下暂停所有积分操作。

### 3.1 验证正常运行

> ⚠️ 如果步骤 2.8 移除了 Account 1，需先回到管理端重新添加。

1. 切换到商户页面 `http://localhost:3000/merchant.html`
2. MetaMask 使用 **Account 1（商家）**
3. 向用户 A 发放 10 LYL → 确认成功
4. 切换到用户页面，用户 A 向用户 B 转账 5 LYL → 确认成功

### 3.2 管理员暂停合约

1. 打开管理页面 `http://localhost:3000/admin.html`
2. MetaMask 切换到 **Account 0（管理员）**
3. 在 **"暂停/恢复合约"** 区域，点击 **"暂停合约"**
4. MetaMask 弹出确认 → 确认交易
5. 页面显示 **"合约已暂停"**

### 3.3 验证暂停效果

1. 切换到商户页面，尝试发放积分 → **交易失败**（合约已暂停）
2. 切换到用户页面，尝试转账 → **交易失败**（合约已暂停）
3. 切换到积分商城 shop.html → 下单正常（链下操作不受影响）
4. 切换到兑换页面 redeem.html → 尝试兑换 → **交易失败**（spend 被暂停）

**预期结果：** 暂停期间所有 on-chain 操作（reward / spend / transfer）均被拒绝，链下功能不受影响。

### 3.4 管理员恢复合约

1. 回到管理页面，点击 **"恢复合约"**
2. MetaMask 确认后，合约恢复正常运行
3. 再次验证：发放积分、转账、兑换均恢复正常

**预期结果：** 恢复后所有功能正常，数据不受影响。

---

**关键机制：**
- **Reward (mint)**：商家发放积分 = 铸币，总供应量增加
- **Spend (burn)**：商家扣除积分 = 销毁，总供应量减少（模拟用户用积分兑换商品）
- **Transfer**：用户间转账 = ERC-20 标准转账，总供应量不变
- **链下订单**：商城购物创建 localStorage 订单，商家确认后触发链上 reward
- **积分兑换**：用户选择奖品 → 确认对话框 → 链上 redeemTokens() 销毁积分
- 所有交易可在 MetaMask 历史记录查看，区块数据可在 Ganache 中查验

## 步骤四：扩展功能演示

### 4.1 水龙头 — 领取测试积分

1. 打开 `http://localhost:3000/faucet.html`
2. MetaMask 切换到 **Account 5（新用户，无积分）**
3. 点击 **"连接 MetaMask"**
4. 确认页面显示 LYL 余额为 0
5. 点击 **"💧 领取 5 LYL"**
6. MetaMask 弹出确认 → 确认交易
7. 页面显示"领取成功"，余额变为 5 LYL
8. 再次点击领取 → 显示"冷却中..."，按钮被禁用

**预期结果：** 新用户可免费获得测试积分，冷却机制生效。每日每地址限领一次。

---

### 4.2 商家间结算

1. 打开 `http://localhost:3000/merchant.html`
2. MetaMask 切换到 **Account 1（商家A）**
3. 下拉到 **"🤝 商家间结算"** 区域
4. 输入 **Account 2（商家B）** 的地址（需先在管理端将 Account 2 设为商家）
5. 输入结算数量：`50`
6. 点击 **"结算转账"**
7. MetaMask 确认后，商家A 的 LYL 余额减少 50，商家B 的 LYL 余额增加 50

**预期结果：** 联盟内商家可相互结算积分，无需通过管理员。目标必须是注册商家。

---

### 4.3 积分汇率设置

1. 在商家后台，下拉到 **"💱 积分汇率设置"** 区域
2. 确认显示 **"当前汇率：未设置（默认 10）"**
3. 输入新汇率：`200`（即 1 元 = 200 积分）
4. 点击 **"设置汇率"**，MetaMask 确认
5. 页面显示 **"当前汇率：1 元 = 200 积分"**

**预期结果：** 商家可自定义积分兑换比例，不同商家可有不同汇率。此汇率信息可用于商城前端展示。

---

### 4.4 积分过期机制演示（可选）

1. 打开管理后台 `http://localhost:3000/admin.html`
2. MetaMask 使用 **Account 0（管理员）**
3. 在管理后台可看到积分有效期设置相关操作
4. 用户可以查看自己的最后活动时间

> 💡 积分过期机制为后台自动运行，演示时可通过合约调用 `checkAndBurnExpired(address)` 手动触发过期检查。默认有效期 365 天，管理员可通过 `setExpiryTime()` 修改。

**预期结果：** 超过有效期未活动的积分可被任何人触发销毁，活动更新后可延长有效时间。
