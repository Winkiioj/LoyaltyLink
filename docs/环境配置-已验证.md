# LoyaltyLink 环境配置指南（已验证版本）

本文档基于实际部署过程更新，包含所有必要步骤和问题解决方案。

---

## 一、前置条件

- **操作系统**：Windows 10/11 或 macOS 或 Linux
- **网络**：可访问 npm 官方源和 GitHub（或配置镜像源）
- **浏览器**：Chrome、Firefox（用于 MetaMask 和 Remix）
- **Git**：版本 2.x 或以上（**重要**）

### ⚠️ 重要：提前安装 Git

**在执行任何 npm install 之前，请先安装 Git**，因为某些包依赖通过 Git 安装。

- **Windows**：访问 https://git-scm.com/download/win 下载并安装
- **安装完成后**重新启动 PowerShell 或命令行窗口
- **验证**：运行 `git --version` 确保成功

## 二、Ganache 安装与启动（成员1）

Ganache 是"个人区块链"，用于在本地模拟以太坊网络。

### 2.1 下载安装

访问Ganache官网：**https://www.trufflesuite.com/ganache**，下载对应操作系统的安装包。

- **Windows**：下载 `.exe` 安装文件，双击运行
- **macOS**：下载 `.dmg` 文件，拖入 Applications 文件夹
- **Linux**：下载 `.AppImage` 文件，赋予执行权限后运行

### 2.2 启动 Ganache 并创建私有链

1. 打开 Ganache 应用程序，点击 **"QUICKSTART"** 按钮
2. Ganache 将自动启动一个本地以太坊私有网络，监听 **`http://127.0.0.1:7545`**
3. 点击右上角的**齿轮图标（Settings）**，配置以下参数：
   - **RPC Server**：`127.0.0.1:7545`（默认）
   - **Network ID**：`1337`（重要，与后续配置一致）
4. 点击 **"Save and Restart"**

### 2.3 记录关键信息

启动成功后，记录以下信息：

- **10 个测试账户**：每个账户初始拥有 100 ETH
- **助记词（Mnemonic）**：点击右上角的钥匙图标 🔑 查看
- **RPC 服务器地址**：`http://127.0.0.1:7545`

**共享给其他成员**：助记词、RPC 端口（7545）、Network ID（1337）

## 三、MetaMask 安装与配置

MetaMask 是一个浏览器钱包插件，用于管理账户、签名交易。

### 3.1 安装 MetaMask

1. 打开 Chrome 或 Firefox 浏览器
2. 访问对应商店搜索 "MetaMask"
3. 点击"添加至浏览器"
4. 安装完成后，浏览器右上角会出现小狐狸图标

### 3.2 创建钱包

1. 点击小狐狸图标 → "开始使用"
2. 选择"创建钱包"
3. 设置密码（用于解锁插件）
4. 记录并保存助记词（**不要截图**，手写记录）

### 3.3 连接 Ganache 私有网络

1. 点击 MetaMask 右上角的网络下拉菜单
2. 选择"添加网络"
3. 填入以下信息：

| 字段            | 值                          |
| :-------------- | :-------------------------- |
| 网络名称        | Ganache Local               |
| 新的 RPC URL    | http://127.0.0.1:7545       |
| 链 ID           | 1337                        |
| 货币符号        | ETH                         |
| 区块浏览器 URL  | （留空）                    |

4. 点击"保存"

### 3.4 导入 Ganache 账户

1. 在 Ganache 中，点击任意账户右侧的钥匙图标 🔑，复制**私钥（Private Key）**
2. 在 MetaMask 中，点击右上角账户图标 → "导入账户"
3. 账户类型选择"私钥"，粘贴私钥
4. 点击"导入"

验证：MetaMask 应显示该账户，余额为 100 ETH

## 四、Node.js 与 npm 安装

### 4.1 安装 Node.js 和 npm

访问 Node.js 官网：**https://nodejs.org**

- 下载 **LTS 版本**（推荐 v18.x 或 v20.x）
- 双击安装文件，一路"Next"（npm 会自动安装）

### 4.2 验证安装

打开 PowerShell 或命令行，运行：

```bash
node --version
# 预期输出：v18.x.x 或 v20.x.x 或更高

npm --version
# 预期输出：9.x.x 或 10.x.x 或更高
```

确认版本号输出正确。

## 五、Hardhat 项目初始化

Hardhat 是以太坊智能合约的开发框架。

### 5.1 创建项目目录

在你选择的工作目录下（如 `D:\Projects`），运行：

```bash
mkdir loyaltylink
cd loyaltylink
```

### 5.2 初始化 npm 项目

```bash
npm init -y
```

### 5.3 安装 Hardhat

```bash
npm install --save-dev hardhat
```

### 5.4 初始化 Hardhat 项目

```bash
npx hardhat --init
```

按照提示进行配置：

| 问题 | 选择 |
| :--- | :--- |
| "Where would you like to initialize the project?" | `.`（当前目录） |
| "What type of project would you like to initialize?" | **A TypeScript Hardhat project using Mocha and Ethers.js ** |
| "Do you want to add a .gitignore?" | `y` |
| "Do you want to install dependencies?" | `y` |

> ⚠️ **注意**：如果安装依赖时出现 Git 相关错误，请确保已先安装 Git（见第一节）。

### 5.5 补充安装依赖包

如果初始化过程中跳过了自动安装，请手动运行：

```bash
npm install --save-dev @nomicfoundation/hardhat-ethers @nomicfoundation/hardhat-chai-matchers ethers chai
```

**重要**：使用 `@nomicfoundation` 而非 `@nomiclabs`，因为后者与 Hardhat 3.x 不兼容。

## 六、配置 hardhat.config.ts

编辑项目根目录下的 `hardhat.config.ts` 文件，将其内容替换为：

```typescript
import { HardhatUserConfig } from "hardhat/config";
import "@nomicfoundation/hardhat-ethers";

const config: HardhatUserConfig = {
  solidity: "0.8.20",
  networks: {
    ganache: {
      url: "http://127.0.0.1:7545",
      chainId: 1337,
      accounts: {
        mnemonic: "你的Ganache助记词（12个单词用空格分隔）"
        // 示例：mnemonic: "test test test test test test test test test test test junk"
      }
    }
  }
};

export default config;
```

**关键步骤**：

1. 从 Ganache 中复制你的助记词（右上角钥匙图标 🔑）
2. 替换配置中的 `"你的Ganache助记词..."`
3. 保存文件

## 七、环境验证

完成上述配置后，验证所有工具能正常协同工作。

### 7.1 验证 Ganache 与 MetaMask 连接

1. 确保 Ganache 正在运行（显示 "RUNNING" 状态）
2. 在 MetaMask 中切换到 "Ganache Local" 网络
3. 检查账户余额是否显示为 **100 ETH**

✅ **验证通过**：账户余额显示正确

### 7.2 验证 Remix IDE 与 Ganache 连接

1. 访问 **https://remix.ethereum.org**
2. 在左侧找到 **"ENVIRONMENT"** 部分
3. 点击环境下拉菜单，选择 **"Custom - External Http Provider"**
4. 在弹出窗口中输入 RPC 地址：`http://127.0.0.1:7545`
5. 点击 "OK"

✅ **验证通过**：Remix 左侧显示 Ganache 账户列表

### 7.3 验证 Hardhat 与 Ganache 连接

在项目目录 `loyaltylink` 中运行：

```bash
npx hardhat compile
```

预期输出：

```
Downloading solc 0.8.20
No contracts to compile（或类似消息）
```

✅ **验证通过**：编译器成功下载，配置正确

## 八、常见问题与解决方案

### 问题 1：MetaMask 无法连接 Ganache

**可能原因**：RPC URL 或链 ID 配置错误

**解决方案**：
- 检查 MetaMask 配置中的 RPC URL 是否为 `http://127.0.0.1:7545`
- 检查链 ID 是否为 `1337`
- 确保 Ganache 处于运行状态

### 问题 2：npm install 出现 Git 相关错误

**可能原因**：系统未安装 Git 或 Git 未在 PATH 中

**解决方案**：
- 访问 https://git-scm.com/download/win 下载并安装 Git
- 安装后**重新启动 PowerShell**
- 验证：`git --version`
- 重新运行 `npm install`

### 问题 3：Remix 连接失败

**可能原因**：Ganache 未运行或防火墙限制

**解决方案**：
- 确认 Ganache 已启动
- 检查防火墙是否允许本地 127.0.0.1:7545 端口
- 尝试在浏览器地址栏直接访问 `http://127.0.0.1:7545`

### 问题 4：Hardhat 编译失败

**可能原因**：Solidity 版本不匹配

**解决方案**：
- 检查 `hardhat.config.ts` 中的 Solidity 版本是否为 `0.8.20`
- 对于合约文件，确保 `pragma solidity` 版本一致

### 问题 5：npm install 依赖冲突

**可能原因**：包版本不兼容

**解决方案**：
- 使用 `@nomicfoundation` 包而非 `@nomiclabs`
- 如有持续问题，运行 `npm install --legacy-peer-deps`
