### 👤 成员2：智能合约开发（核心） 

**目标**：编写 `LoyaltyToken.sol`，继承 OpenZeppelin ERC-20，添加商家管理功能。

**第2周：编写合约代码**

1. **在 `loyaltylink/contracts/` 下创建 `LoyaltyToken.sol`**：

solidity

```
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.20;

import "@openzeppelin/contracts/token/ERC20/ERC20.sol";
import "@openzeppelin/contracts/access/Ownable.sol";

contract LoyaltyToken is ERC20, Ownable {
    mapping(address => bool) public merchants;
    
    event MerchantAdded(address indexed merchant);
    event MerchantRemoved(address indexed merchant);
    event Reward(address indexed fromMerchant, address indexed toUser, uint256 amount);
    event Spend(address indexed fromMerchant, address indexed fromUser, uint256 amount);

    constructor(string memory name, string memory symbol) 
        ERC20(name, symbol) 
        Ownable(msg.sender) 
    {}

    modifier onlyMerchant() {
        require(merchants[msg.sender], "Not a merchant");
        _;
    }

    function addMerchant(address merchant) external onlyOwner {
        merchants[merchant] = true;
        emit MerchantAdded(merchant);
    }

    function removeMerchant(address merchant) external onlyOwner {
        merchants[merchant] = false;
        emit MerchantRemoved(merchant);
    }

    function reward(address user, uint256 amount) external onlyMerchant {
        _mint(user, amount);
        emit Reward(msg.sender, user, amount);
    }

    function spend(address user, uint256 amount) external onlyMerchant {
        require(balanceOf(user) >= amount, "Insufficient balance");
        _burn(user, amount);
        emit Spend(msg.sender, user, amount);
    }
}
```

### 👤 成员3：部署配置

**目标**：编写部署脚本，将合约部署到 Ganache，管理 ABI 和地址。

**第2周：编写部署脚本**

1. **在 `ignition/modules/` 下创建部署模块**（Hardhat Ignition 方式）：

   typescript

   ```
   // ignition/modules/LoyaltyToken.ts
   import { buildModule } from "@nomicfoundation/hardhat-ignition/modules";
   
   const LoyaltyTokenModule = buildModule("LoyaltyTokenModule", (m) => {
     const token = m.contract("LoyaltyToken", ["LoyaltyLink", "LYL"]);
     return { token };
   });
   
   export default LoyaltyTokenModule;
   ```

   

2. **配置 `hardhat.config.ts`**（确保 ganache 网络配置正确）：

   typescript

   ```
   import { HardhatUserConfig } from "hardhat/config";
   import "@nomicfoundation/hardhat-toolbox";
   
   const config: HardhatUserConfig = {
     solidity: "0.8.20",
     networks: {
       ganache: {
         url: "http://127.0.0.1:7545",
         chainId: 1337,
         accounts: {
           mnemonic: "你的助记词（12个单词）"
         }
       }
     }
   };
   
   export default config;
   ```

   

**第3周：部署到 Ganache**

1. 启动 Ganache（QuickStart）

2. 运行部署命令：

   bash

   ```
   npx hardhat ignition deploy ./ignition/modules/LoyaltyToken.ts --network ganache
   ```

   

3. 成功后会输出合约地址，**复制并保存**（例如 `0x1234...`）。

4. 导出 ABI：

   - 在 `artifacts/contracts/LoyaltyToken.sol/LoyaltyToken.json` 中找到 `abi` 数组。
   - 将其复制到一个新的文件 `frontend/abi.json` 中，供前端使用。

**第4周：ABI 版本管理**

- 每次合约更新后，重新部署并更新前端 `abi.json`。
- 准备演示账户：记录 Ganache 中前三个账户地址（管理员、商家1、商家2）。

**第3周：连接合约 + 余额查询**

- 编写 `app.js`：

javascript

```
let web3;
let contract;
let contractAddress = "0x..."; // 由成员3提供
let abi; // 从 abi.json 加载

window.addEventListener('load', async () => {
    if (window.ethereum) {
        web3 = new Web3(window.ethereum);
        await window.ethereum.request({ method: 'eth_requestAccounts' });
        contract = new web3.eth.Contract(abi, contractAddress);
        await updateBalance();
    }
});

async function updateBalance() {
    const accounts = await web3.eth.getAccounts();
    const balance = await contract.methods.balanceOf(accounts[0]).call();
    document.getElementById('balance').innerText = `余额: ${balance} LYL`;
}

async function transfer() {
    const to = document.getElementById('toAddress').value;
    const amount = document.getElementById('transferAmount').value;
    const accounts = await web3.eth.getAccounts();
    await contract.methods.transfer(to, amount).send({ from: accounts[0] });
    await updateBalance();
}
```

## 四、常见问题预防

| 问题                   | 解决方案                                                     |
| :--------------------- | :----------------------------------------------------------- |
| 合约部署后前端调用报错 | 检查 ABI 是否正确，合约地址是否最新，网络是否为 Ganache      |
| MetaMask 无法切换账户  | 点击右上角账户图标 → “创建账户” 或 “导入账户”（使用 Ganache 私钥） |
| 交易一直 pending       | 检查 Gas 价格是否足够（Ganache 默认自动设置）                |
| 前端连接后余额为 0     | 确认调用的合约方法名是否正确（`balanceOf`），地址是否正确    |
