# 组员拉取与本地运行指南

## 前置要求

| 工具 | 版本要求 | 下载地址 |
|------|---------|---------|
| Node.js | v18+ | https://nodejs.org |
| Git | 任意版本 | https://git-scm.com |
| MetaMask | 浏览器插件 | Chrome 应用商店 |
| Ganache | GUI 或 CLI | https://trufflesuite.com/ganache |

---

## 一、拉取代码

```bash
# 1. 克隆仓库
git clone https://github.com/Winkiioj/LoyaltyLink.git

# 2. 进入项目目录
cd LoyaltyLink

# 3. 切换到对应分支
git checkout feature/switch-to-eth

# 4. 安装依赖
npm install
```

---

## 二、启动区块链网络

### 方式 A：使用 Ganache GUI（推荐）

1. 打开 Ganache
2. 点击 **"Quickstart (Ethereum)"**
3. 点击右上角齿轮图标 → **"Accounts & Keys"**
4. 将 **"Mnemonic"** 设置为：

```
maid notable twist mutual dune speed come dolphin wet gaze scout sort
```

5. 点击 **"Save & Restart"**
6. 确认 RPC 地址为 `http://127.0.0.1:7545`

### 方式 B：使用 Ganache CLI

```bash
# 全局安装 ganache
npm install -g ganache

# 启动（使用指定的助记词）
ganache --mnemonic "maid notable twist mutual dune speed come dolphin wet gaze scout sort"
```

## 三、启动前端

```bash
# 在项目根目录下执行
npx http-server frontend/ -p 3000
```

启动后访问：
- 用户端：`http://localhost:3000/index.html`
- 商户端：`http://localhost:3000/merchant.html`

---

## 四、配置 MetaMask

### 4.1 添加 Ganache 网络

1. 点击 MetaMask → 网络切换 → **"添加网络"**
2. 填写以下信息：

| 字段 | 值 |
|------|-----|
| 网络名称 | `Ganache Local` |
| RPC URL | `http://127.0.0.1:7545` |
| Chain ID | `1337` |
| 货币符号 | `ETH` |

3. 点击 **"保存"**

### 4.2 导入测试账户

Ganache 默认提供 10 个账户，每个有 100 ETH。

1. 在 Ganache 中点击任意账户右侧的 **钥匙图标** 复制私钥
2. 在 MetaMask 中：点击头像 → **"导入账户"** → 粘贴私钥
3. **建议导入 2 个账户**：一个作为商户，一个作为用户

---

## 五、验证环境

1. **检查 MetaMask**：确认已连接到 "Ganache Local" 网络，且账户余额显示 ~100 ETH
2. **打开用户端** `http://localhost:3000/index.html` → 点击 **"连接 MetaMask"** → 应显示余额
3. **打开商户端** `http://localhost:3000/merchant.html` → 点击 **"连接 MetaMask"** → 应显示余额

## 六、常见问题

### Q：`npm install` 报错

确保 Node.js 版本 ≥ 18，并尝试：

```bash
# 删除 node_modules 后重试
rm -rf node_modules
npm install
```

### Q：连接 MetaMask 时没有弹出确认窗口

- 确保 MetaMask 已解锁
- 确保 MetaMask 已连接到 Ganache 网络（Chain ID 1337）
- 刷新页面后重试

### Q：交易失败 / 余额不足

- 确认使用的账户在 Ganache 中有 ETH 余额
- 确认 MetaMask 中当前选中的账户正确

### Q：前端页面无法打开

- 确认 `http-server` 已启动且端口 `3000` 未被占用
- 可以尝试更换端口：`npx http-server frontend/ -p 8080`
