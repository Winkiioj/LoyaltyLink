# 第 3 页 — 项目背景与痛点

## 页面标题
传统积分体系的困境

## 页面左侧文字

痛点一：积分碎片化
用户在不同商家积累积分，互不相通。咖啡店积分、超市积分、航空里程……积分散落各处，无法集中使用。

痛点二：积分贬值风险
商家可单方面修改规则——调整兑换比例、设置有效期、甚至清零。积分只是商家数据库里的一行记录，用户没有控制权。国内每年约 30% 积分因过期浪费。

痛点三：流通困难
积分不能转让、不能赠送。家人之间也无法共享。中小商户自建积分系统年均成本 2-5 万元，积分核销率不足 20%。

# 第 7 页 — 智能合约设计

## 页面标题
核心智能合约：LoyaltyToken.sol

## 页面左侧（代码展示）
展示以下核心代码（等宽字体，黑底白字或语法高亮）：

```solidity
contract LoyaltyToken is ERC20, Ownable, Pausable {
    mapping(address => bool) public merchants;

    modifier onlyMerchant() {
        require(merchants[msg.sender], "Not a merchant");
        _;
    }

    // 商家发放积分（铸币）———— 总量增加
    function reward(address user, uint256 amount)
        external onlyMerchant whenNotPaused {
        _mint(user, amount);
        emit Reward(msg.sender, user, amount);
    }

    // 商家扣除积分（销毁）———— 总量减少
    function spend(address user, uint256 amount)
        external onlyMerchant whenNotPaused {
        require(balanceOf(user) >= amount, "Insufficient balance");
        _burn(user, amount);
        emit Spend(msg.sender, user, amount);
    }

    // 用户自主兑换（自毁积分）———— 总量减少
    function redeemTokens(uint256 amount)
        external whenNotPaused {
        _burn(msg.sender, amount);
        emit TokensRedeemed(msg.sender, amount);
    }

    // 暂停 / 恢复（紧急控制）
    function pause() external onlyOwner { _pause(); }
    function unpause() external onlyOwner { _unpause(); }
}
```

## 页面右侧（设计要点说明）

要点一：三重继承
ERC20（标准代币）+ Ownable（所有权管控）+ Pausable（紧急暂停）。全部继承自 OpenZeppelin 审计级库，安全可靠。

要点二：商家白名单
mapping(address => bool) 存储权限。只有 Owner 可添加/移除。只有白名单商家可调用 reward 和 spend。

要点三：三种积分操作
- reward() = _mint() 铸币 → 总量增加
- spend() = _burn() 销毁 → 总量减少
- redeemTokens() = _burn() 自毁 → 总量减少，任何人可用

要点四：Pausable 紧急控制
Owner 可随时暂停合约，暂停时 reward/spend/redeemTokens/transfer 全部锁定，但 addMerchant/removeMerchant 不受影响。用于紧急漏洞修复。

要点五：事件完备
8 种事件：MerchantAdded、MerchantRemoved、Reward、Spend、TokensRedeemed、ContractPaused、ContractUnpaused（以及 ERC-20 标准的 Transfer、Approval）

# 第 18 页 — 用户体验增强特性

## 页面标题

## 页面正文（六个特性卡片，3×2 网格排列）

卡片一：确认对话框 (LL.confirm)
所有涉及资金的操作（转账、发放、扣除、兑换、暂停）都有二次确认弹窗。通过 JS 动态创建 overlay + 对话框 DOM，点击外部或取消按钮可关闭。防止误操作。

卡片二：按钮加载状态 (LL.setButtonLoading)
点击后按钮自动禁用，文案变为"处理中..."并显示旋转动画。防止用户重复提交交易。交易完成或失败后自动恢复。

卡片三：字段级错误提示 (LL.showFieldError)
每次输入验证失败，在对应输入框下方显示红色错误文字。输入时自动清除。比 alert() 体验更好，精确定位问题。

卡片四：中文错误翻译 (LL.translateError)
Solidity revert 消息（英文）自动翻译为用户友好的中文提示。例如："LoyaltyToken: Not a merchant" → "您不是商家，无权执行此操作"。降低非技术用户的理解门槛。

卡片五：角色门控 (checkPageRole)
未授权用户打开管理端/商户端时，不是白屏或报错，而是显示友好的权限提示页面。角色要求写在 data-page 属性中，统一由 LL 模块处理。

卡片六：导航栏 + 当前页高亮 (LL.setActiveNav)
六个页面共享统一的导航栏。当前页面自动高亮。用户可在积分钱包、商城、兑换、商户管理、历史记录、管理后台之间自由切换。
